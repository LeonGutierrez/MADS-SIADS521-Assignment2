What is Python Seaborn library?

Seaborn is a tabular/declarative plotting library. Plotting libraries like Seaborn, can treat columns of data frames as properties of a graph at a high level. Seaborn is a python library that is used to create graphics in Python. It is built on top of matplotlib and integrated with pandas, scypy, and numpy. This package is used to simplify complex data visualizations. Some of the features Seaborn offers are:

  • An API that is dataset-oriented to allow comparison between multiple variables
  • Supports multi-plot grids allowing the user to build complex visualizations
  • Univariate and bivariate charts to compare between subsets of data
  • Availability of different color palettes to better highlight variable patterns
  • Estimates and plots linear regression automatically for different types of dependent vaiables
  • Availability of different color palettes to better highlight variable patterns

Seaborn aims to make visualization a central part of exploring and understanding data. Its dataset-oriented plotting functions operate on dataframes and arrays containing whole datasets and internally perform the necessary semantic mapping and statistical aggregation to produce informative plots.

In summary, I chose Seaborn for this tutorial because Seaborn is very simple to use compare to other packages like Matplotlib. I makes hard visualizations easy to to be defined. I personally like that it automatically displays linear regressions when I analyze scatterplots

Seaborn - Strengths and Weaknesses

    Strengths:

    • Good for style themes right off the box
    • Allows for custom color palettes. A list can be created using color codes from https://bulbapedia.bulbagarden.net/wiki/Category:Type_color_templates. Once created, the palette='newpalette' can be used
    Weaknesses:

    • Dificult to manipulate dataframes. Easier to pre-format the dataset before it is used in Seaborn
    • Library is relatively low-level
    • Defaults are not optimal in my opinion (no gridlines, white background, etc.)

Why using scatterplots to analyze bivariate relationships?

In general, I think scatterplot are a great way to explore your data when you want to find relationships in your data. That relationship can be calculated as a scalar (Pearson or Spearman). However, a visual exploration provides insights not capture by a single number. This exploratory analysis is very useful in cases where you want to predict a value, so one variable's final estimation will depend on the value of another one. In summary, this types of analysis are good in cases:

  1. When you want to analize paired numerical data.
  2. When trying to determine whether the variables you are analyzing are correlated. This correlation helps determine whether one variable could be used to explain the behavior of the other
  3. In classification and regression problems, scatterplots allow you to see these correlations in a way that it is easy to determine explanatory power

Data Set used in this Tutorial

This dataset is related to red wine variants of the Portuguese "Vinho Verde" wine. This dataset has been used in multiple websites to serve as an example for regression models. For this reason, I think it was a good database for this tutorial. This data is open source and can be downloaded from multiple places. For more details, consult the reference [Cortez et al., 2009]. Due to privacy and logistic issues, only physicochemical (inputs) and sensory (the output) variables are available (e.g. there is no data about grape types, wine brand, wine selling price, etc.).

This dataset is also available from the UCI machine learning repository,

https://archive.ics.uci.edu/ml/datasets/wine+quality

Quality, is a score between 0 and 10 based on sensory responses. All other variables are based on physicochemical tests.

I decided to use this database not only because it has been widely used to demostrate regression models but because access is easy from multiple locations. Additionally, the data contains sufficient information (1,599 observations)to use it in this context.

Relevant publication: P. Cortez, A. Cerdeira, F. Almeida, T. Matos and J. Reis. Modeling wine preferences by data mining from physicochemical properties. In Decision Support Systems, Elsevier, 47(4):547-553, 2009.
